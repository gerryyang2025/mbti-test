# Security Policy

## Supported Version

当前仅默认支持仓库默认分支上的最新版本。

## Reporting a Vulnerability

- 请不要在公开 Issue 中直接披露可被利用的敏感细节。
- 如果仓库已启用 GitHub Security Advisories，优先使用私密安全报告渠道。
- 如果暂时没有私密报告入口，请先提交不包含利用细节的简要说明，并请求维护者提供后续沟通方式。

## Additional Notes

- 请勿在问题报告中附带个人隐私信息、真实答题记录或其他敏感数据。
- 与 UI 展示、文案错误、静态资源加载失败相关的问题，通常更适合直接提交普通 Issue。
